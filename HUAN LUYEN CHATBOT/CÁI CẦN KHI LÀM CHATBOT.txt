####  Các thay đổi quan trọng bạn cần biết:
1. Quản lý API Key Pro (Siêu Admin):

Trong tab 🔑 API Keys của Super Admin, mình đã thêm ô: 🌟 API Key Trả Phí.
Bạn hãy nhập API Key "xịn" (đã nạp tiền) vào đây. Chỉ những người dùng đã nâng cấp lên PRO mới được dùng Key này.
2. Nâng cấp PRO cho Shop:

Tại danh sách cửa hàng, mình đã thêm cột Gói Dịch Vụ. Cửa hàng bình thường sẽ hiện chữ FREE.
Có một biểu tượng chìa khóa vàng kế bên mỗi cửa hàng. Khi bạn nhấn vào đó, hệ thống sẽ hỏi số ngày muốn nâng cấp (VD: 30 ngày).
Sau khi bấm OK, cửa hàng đó sẽ lập tức biến thành 🌟 PRO. Hệ thống sẽ tự động gán ngày hết hạn.
3. Tự động hóa & Thông minh:

Ưu tiên API: Khi một User thuộc shop PRO gửi tin nhắn, hệ thống sẽ ưu tiên dùng API Key Trả Phí của bạn để trả lời nhanh và ổn định nhất.
Tự động xuống hạng: Khi hết thời gian PRO, hệ thống sẽ tự động chuyển shop đó về gói FREE và dùng lại API Key thường mà không cần bạn phải thao tác tay.
Icon Pro Chuyên nghiệp: Trên Dashboard của User, nếu họ là PRO, sẽ hiện một "Huy hiệu 🌟 PRO" nhấp nháy màu vàng cực kì sang trọng, giúp họ cảm nhận được giá trị khi bỏ tiền nâng cấp.  #####

  #### Mình đã hoàn thiện cơ chế Giám sát API Key thông minh theo đúng yêu cầu của bạn:

Các điểm cải tiến mới:
Báo cáo tên Key bị lỗi:

Trong bảng điều khiển Nhật ký lỗi, bây giờ bạn sẽ thấy rõ tên Key bị lỗi: [Key 1], [Key 2] hoặc [Key PRO].
Hệ thống sẽ ghi rõ nguyên nhân: API key bị sai hoặc hư, Kết nối thất bại, hay Bị quá tải (429).
Cơ chế Fallback PRO -> FREE tự động:

Nếu khách hàng là gói PRO nhưng API Key PRO của bạn gặp vấn đề (sai key, hết hạn, hoặc bị Google báo quá tải), hệ thống sẽ không trả về lỗi cho khách ngay.
Thay vào đó, nó sẽ âm thầm chuyển sang dùng các Key Free dự phòng (Key 1, Key 2) để phục vụ khách tiếp tục cuộc trò chuyện.
Đồng thời, một cảnh báo sẽ được gửi về tab ⚠️ Nhật ký Lỗi cho bạn với nội dung: "Key PRO đang bị quá tải/lỗi. Hệ thống đã tự chuyển về Key Free cho khách."
Thông minh và Tiết kiệm:

Hệ thống sẽ tự động thử các Key theo thứ tự ưu tiên: Key PRO -> Key 1 -> Key 2.
Nếu phát hiện Key nào bị sai (Invalid), nó sẽ bỏ qua Key đó ngay lập tức để tiết kiệm thời gian chờ cho khách.
Bây giờ bạn có thể hoàn toàn yên tâm ngồi theo dõi hệ thống, nếu có bất kỳ Key nào "có vấn đề", bạn sẽ biết chính xác là cái nào để vào thay thế kịp thời! ✅_    ######
