Dưới đây là ý tưởng về cách script này hoạt động:

1. Quy trình của Script (Concept)
Script này sẽ sử dụng chính API Gemini để "đọc hiểu" và "viết lại" dữ liệu của bạn theo 3 bước:

Bước 1 - Lọc nhiễu: Loại bỏ các thông tin thừa, các câu từ lặp lại không cần thiết trong file gốc.
Bước 2 - Chuyển đổi Q&A: Biến các đoạn mô tả khô khan thành các cặp Câu hỏi - Câu trả lời (FAQ) theo phong cách trò chuyện.
Bước 3 - Thổi hồn (Emotional Rewrite): Chỉnh sửa văn phong của toàn bộ dữ liệu sang kiểu "ấm áp, tình cảm" như bạn mong muốn.
2. Cấu trúc đầu ra (Output chuẩn)
Thay vì một file text dài, script sẽ xuất ra một định dạng cực kỳ dễ dùng cho Dashboard, ví dụ:

json
{
  "product_info": "Tóm tắt 5-10 dòng tinh túy nhất về sản phẩm, lồng ghép sự trân trọng.",

    {
      "q": "Khách hỏi về giá...",
      "a": "Trả lời khéo léo, nhấn mạnh vào giá trị sức khỏe và tình thân thay vì con số."

      "a": "Lời an ủi và cam kết đồng hành nhẹ nhàng."
    }

}

Bạn có thể tạo một file standardize.js trong thư mục gốc. Khi chạy, nó sẽ đọc file FAQ.txt và trả về kết quả đã được "tối ưu hóa":

javascript
// Ý tưởng script (Bạn có thể chạy thử nếu muốn)
const fs = require('fs');
// Gọi API Gemini để xử lý văn bản
async function standardize(fileName) {
  const rawData = fs.readFileSync(fileName, 'utf-8');
  
  const prompt = `
    Dưới đây là dữ liệu thô của shop. Hãy giúp mình:
    1. Trích xuất các ý chính về sản phẩm (Product Info).
    2. Chuyển các thông tin khác thành bộ FAQ (Câu hỏi - Trả lời).
    3. QUAN TRỌNG: Hãy viết lại toàn bộ bằng văn phong cực kỳ TÌNH CẢM, ẤM ÁP, chân thành.
    
    Dữ liệu thô: ${rawData}
  `;
  
  // Gửi prompt này lên Gemini và nhận về kết quả đã "chuẩn hóa"
  // Sau đó bạn chỉ việc copy results dán vào Dashboard.
}
